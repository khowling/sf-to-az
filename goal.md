
Build a single-tenant version of salesforce crm, using open-source technologies, but still allowing a extensible schema configuration driven from a end user config experience.


The Database should be Postgres, and app in Typescript, with React frontend, using the best framework for this type of app, consider Zod and Tailwind.

Just build out the Account, Contact and Opportunity objects for now.

Allow users to add fields to these objects, using a business definition of the field types, validations, and required status, and also allow the user to place the field in the object page layout.  Map this to the appropriate postgres table and fields.


You are only finished when you have a working website with a database backend that can add new opportunities and Accounts, and with one of the Opportunity fields looking up to the Account
